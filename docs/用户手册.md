# 智能数仓建设平台 — 用户手册

## 这是什么

用 Claude Code 对话来建数据仓库的工具包。

下载、解压、装依赖、打开 Claude、说话就行了。Claude 自动调命令、生成代码、跑 dbt、出看板。

---

## 第一步：安装

```bash
pip install -e .
```

依赖会全部自动装好：dbt-core、dbt-duckdb、PyYAML 等。

---

## 第二步：配置

### project.yml — 告诉平台你的项目长什么样

```yaml
project:
  name: "我的数据仓库"

source:
  adapter: sap_ecc          # 源系统类型：sap_ecc / mysql / custom

domains:
  - code: order
    name: 订单域
    description: "销售订单、开票、交货"
  - code: fi
    name: 财务域
    description: "费用报销、成本核算"

scenarios:
  - name: 每日开票
    code: daily_billing
    steps:
      - run: "+dwd_order_+"
      - run: "+dws_fi_invoice_+"
      - test: "+dws_fi_+"
```

### profiles.yml — 告诉 dbt 数据库在哪

```yaml
my_warehouse:
  target: dev
  outputs:
    dev:                          # 本地 DuckDB（快速验证）
      type: duckdb
      path: warehouse.duckdb
    hive:                         # 生产 Hive
      type: spark
      method: thrift
      host: 192.168.1.100
      port: 10000
      user: spark
      password: your_password
      schema: dmp_dw
```

## CLI 命令速查

所有命令都可以直接敲，也可以让 Claude 代劳。

### 扫描项目

```bash
dbt-platform scan                  # 看看有多少模型和宏
dbt-platform scan --json           # 机器可读格式
```

### 生成代码

```bash
# 生成一张 DWD 清洗表
dbt-platform dwd generate \
  --table vbrk \
  --columns "erdat:date,netwr:decimal,kunnr:string"

# 直接保存到文件
dbt-platform dwd generate \
  --table vbrk \
  --columns "erdat:date,netwr:decimal" \
  --output models/dwd/order/dwd_order_vbrk.sql
```

列类型：`string` — 字符串 · `date` — 日期 · `decimal` — 数值 · `flag` — 布尔

### 管理 Agent

```bash
dbt-platform agent list            # 8 个 Agent，谁完成了谁在等
dbt-platform agent show agent-03   # 看某个 Agent 的详细定义
dbt-platform agent status          # 所有 Agent 的执行状态

# 执行单个 Agent（直接跑 dbt）
dbt-platform agent run agent-03 --domain fi --target dev

# 管道执行（按依赖顺序自动跑）
dbt-platform agent pipeline --dry-run    # 先看计划
dbt-platform agent pipeline --target dev # 正式执行
```

### 执行场景

```bash
dbt-platform run daily_billing --target dev      # 本地验证
dbt-platform run daily_billing --target hive     # 上生产
dbt-platform run daily_billing --dry-run         # 只看不跑
```

### 数据验证

```bash
dbt-platform validate              # Hive vs OceanBase 行数对比
```

### BI 看板

```bash
dbt-platform bi list
dbt-platform bi generate --config config/bi_dashboards/开票明细.yml --target hive
dbt-platform bi generate-all --target hive
```

生成的 HTML 在 `output/bi/`，浏览器直接打开。

### 项目切换

```bash
dbt-platform project list              # 有哪些项目
dbt-platform project current           # 当前在哪个
dbt-platform project switch benz       # 换一个
dbt-platform project switch .          # 回到默认
dbt-platform init 新项目 --adapter mysql   # 新建
```

### 查看配置

```bash
dbt-platform list                  # 列出所有业务域和场景
```

## 目录说明

```
├── project.yml              ← 你填的项目配置
├── dbt_project.yml          ← dbt 配置（一般不用改）
├── profiles.yml             ← 数据库连接
│
├── models/                  ← SQL 模型
│   ├── dwd/                 ← 明细层（1:1 清洗）
│   ├── dws/                 ← 服务层（多表 JOIN）
│   ├── dim/                 ← 维度层（共享维度）
│   └── ads/                 ← 应用层（BI 就绪）
│
├── macros/                  ← 可复用的 Jinja 宏
├── config/                  ← BI 看板的 YAML 配置
├── output/                  ← 生成的 BI HTML 看板
├── projects/                ← 其他项目存这里
│
├── dbt_platform/            ← 引擎（不用管）
├── .claude/                 ← Claude Code 配置
├── platform/electron/       ← 监控面板（双击启动）
├── scripts/                 ← 独立工具脚本
├── bin/                     ← 一键启动脚本
│
└── pyproject.toml           ← pip install -e .
```

## 8 个 Agent 说明

Agent 是一套标准化的数仓建设流程，Claude Code 按这个流程干活。

| Agent | 做什么 | 依赖 |
|------|------|------|
| agent-00 规范制定 | 定命名规则、编码标准 | — |
| agent-01 场景分析 | 分析需求，推荐表清单 | agent-00 |
| agent-02 ODS搭建 | 盘点源表，关联 sources.yml | agent-01 |
| agent-03 DWD开发 | 1:1 清洗，类型转换，去重 | agent-02 |
| agent-04 DWS开发 | 多表 JOIN，业务宽表 | agent-03 |
| agent-05 DIM设计 | 共享维度表 | agent-03 |
| agent-06 ADS开发 | BI 就绪输出表 | agent-04, agent-05 |
| agent-07 OB迁移验证 | OceanBase 双写验证 | agent-06 |

流程：00 → 01 → 02 → 03 → 04 → 06 → 07，03 同时通向 05。

## 依赖

安装 `pip install -e .` 时自动装好：

- Python ≥ 3.10
- dbt-core ≥ 1.8
- dbt-duckdb（本地开发用）
- dbt-spark（Hive 生产用）
- PyYAML、PyHive、PyMySQL

Claude Code 是可选的——没有也能用 CLI 手动执行所有命令。
