> 版本 2.0 | 2026-07-02 | 通用数据仓库建设平台

## 一、项目概述

智能数仓建设平台是一个 **AI 驱动的通用数据仓库建设系统**。将任意源系统（SAP ECC / MySQL / Oracle / ...）的数据通过 dbt 自动化迁移到目标数仓（Hive / OceanBase / StarRocks / ...），并生成 HTML BI 看板供业务消费。

当前示例项目：**人教社(PEP) SAP ECC → Hive + OceanBase → FineReport BI**。

### 技术栈

| 层 | 技术 |
|---|------|
| 数据建模 | dbt-core 1.11.9 + Jinja2 宏 |
| 目标数仓 | Hive (Spark Thrift) / DuckDB (dev) / OceanBase (BI输出) / MySQL |
| 引擎语言 | Python 3 + JavaScript (Electron) |
| 桌面平台 | Electron + React + Vite + Recharts |
| BI 看板 | ECharts 5.5 + 静态 HTML |
| CI/CD | GitHub Actions + DolphinScheduler |
| 版本管理 | Git (Gitee) |

## 五、数据流

```
SAP ECC (ABAP) → DataX → Hive dmp_ods (只读)
                            ↓ dbt DWD (1:1清洗+去重+7审计列)
                         Hive dmp_dw (DWD + DWS中间表+宽表)
                            ↓ dbt ADS (参数化，BI就绪)
                         Hive dmp_ads + OceanBase dmp_ads (双写)
                            ↓
                    ┌──────┴──────┐
                    │             │
              FineReport      HTML BI 看板
              (外部BI)        (engine/bi/)
```

### 5层数据模型

| 层 | Schema | 职责 | 数量 |
|----|--------|------|------|
| ODS | dmp_ods | 原始数据，只读 | 74 源表 |
| DWD | dmp_dw | 1:1清洗，去重，类型转换 | 86 模型 |
| DWS | dmp_dw | 多表JOIN，业务逻辑 | 8 中间表 |
| DIM | dmp_dim | 共享维度 | 1 表 |
| ADS | dmp_ads | BI就绪，Hive+OB双写 | 1 表 |

### 层间引用规则

| 引用 | 允许 |
|------|------|
| `source('ods')` → `ref('dwd_*')` | ✅ |
| `ref('dwd_*')` → `ref('dws_*')` | ✅ |
| `source('ods')` → `ref('dws_*')` | ❌ (绕过DWD) |
| `ref('ads_*')` → anything | ❌ (ADS是终点) |

## 八、核心设计原则

| 原则 | 实现 |
|------|------|
| **引擎×配置分离** | `engine/` 零业务耦合，`project.yml` 承载所有业务信息 |
| **适配器隔离方言** | `engine/adapters/` — 源系统差异封装在适配器内部 |
| **dbt 是总线** | 所有数据操作通过 `dbt parse/run/test`，不在脚本中私造SQL |
| **7审计列必须** | 每个 DWD/DWS 模型包含 `{{ audit_columns() }}` |
| **只清洗不过滤** | DWD 保留所有行，业务过滤在 DWS |
| **场景驱动开发** | 按 ADS←DWS←DWD←ODS 顺序，一个场景一个场景推进 |
| **平台可复现** | 所有产出可直接复制执行，不依赖会话临时状态 |
