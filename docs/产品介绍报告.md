# 智能数仓建设平台 — 通用产品介绍报告

> 一个 **AI 驱动的通用数据仓库建设平台**，将源系统数据自动化迁移到目标数仓，对接 BI 消费。

---

## 一、产品定位

### 解决什么问题

企业将 ERP/业务系统数据迁移到数据仓库，通常面临：

1. **字段映射混乱** — SAP yyyyMMdd、Oracle DATE、MySQL datetime，每种源系统方言不同
2. **模型重复手写** — 每张表都要人写 ODS→DWD→DWS→ADS SQL，上百张表耗时数月
3. **质量不可验证** — 迁移后不知道丢没丢数据、错没错字段
4. **场景编排失控** — ETL 依赖链复杂，一个场景涉及十几张表，手动执行极易出错
5. **AI 辅助断层** — 大模型能生成 SQL，但没有项目上下文，脱节严重

### 我们怎么解决

```
源系统适配器 → 自动发现字段 → AI 辅助生成模型 → dbt 标准化执行 → 跨库自动验证 → BI 消费
```

**核心思路：引擎 × 配置分离。**

## 四、核心设计：引擎 × 配置分离

通用引擎（代码，可复用）与项目配置（数据，可替换）分离：

| 层次 | 属于引擎（通用） | 属于配置（项目级） |
|------|-----------------|-------------------|
| 类型转换 | `engine/adapters/base.py` — 接口定义 | `project.yml` — `source.adapter: sap_ecc` |
| 去重逻辑 | `DwdGenerator.generate()` — 通用模板 | 适配器的 `dedup_keys()` — SAP 用 mandt+pk，MySQL 用 id |
| 审计列 | `macros/audit_columns.sql` — 7 列固定模板 | 无需配置 |
| SQL 生成 | `engine/generators/dwd_generator.py` — 引擎 | 字段清单 + 列定义（适配器自动发现） |
| 场景编排 | `engine/orchestrator/agent_runner.py` — 执行器 | `project.yml` — `scenarios` 字段 |
| CI/CD | `engine/ci_templates/` — 模板 | 由 `project.yml` 的连接信息渲染 |
| UI 提示 | `SYSTEM_PROMPT` — 动态渲染 | `project.yml` — `project.name` / `project.description` |

### 适配器机制

每种源系统实现统一接口：

```python
class SourceAdapter(ABC):
    def string_to_sql(col, default)  # TRIM + COALESCE
    def date_to_sql(col)              # DATE 转换
    def number_to_sql(col, precision) # NULL → 0 + DECIMAL
    def flag_to_sql(col)              # 布尔标记
    def discover_tables(connection)   # 列出源表
    def discover_columns(connection, table) # 列出列
    def dedup_keys(table)             # 去重主键
    def dedup_order(table)            # 去重排序
```

内置适配器：`sap_ecc`、`mysql`、`custom`。可扩展 `oracle_ebs`、`postgres` 等。

### 配置中心 project.yml

```yaml
project:
  name: "PEP企业数据仓库"     # ← 唯一需要改的地方
source:
  adapter: sap_ecc              # ← 选适配器
warehouse:
  primary: { adapter: hive_spark }
layers: [ODS, DWD, DWS, DIM, ADS]
domains: [order, fi, md, ...]   # ← 定义业务域
scenarios:                      # ← 定义场景 DAG
  - name: 开票明细
    schedule: "0 6 * * *"
    steps:
      - run: "+dwd_order_+"
      - run: "dws_fi_invoice_base"
      - validate: cross_db
```

## 七、工作流

### 新项目启动

```bash
# 1. 初始化
dbt-platform init my_project --adapter sap_ecc

# 2. 编辑 project.yml 填入源系统连接 + 业务域
# 3. 自动发现源表
dbt-platform discover

# 4. AI 辅助生成 DWD 模型（通过 Claude Code 或 gen_dwd_sql.py）
# 5. 验证
dbt-platform validate

# 6. 部署
git push → CI 门禁自动 parse + test → merge master → 自动 run hive
```

### 日常开发

```
Electron 桌面应用
  → 扫描项目 (scan-project)
  → 查看 DAG 血缘 (compute-lineage)
  → 启动 Claude Code 讨论需求
  → 一键执行 dbt run --target dev
  → DuckDB 本地验证通过
  → 提交代码 → CI 自动 parse + test
```

### 生产调度

```
DolphinScheduler
  → 每日 06:00 场景 01（开票明细）
  → 每月 01 日 场景 02（收入执行）
  → 每周一 场景 03（表单报表）
  → 调度失败自动告警（企业微信/飞书）
```
